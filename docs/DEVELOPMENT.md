# Стандарти розробки (Development Guide)

Для підтримки чистоти коду в проекті використовуються **ESLint** та **Prettier**.

## 1. Команди для перевірки коду

### Перевірка на наявність помилок (Linting):

Ця команда перевірить код на дотримання правил React, React Native та TypeScript.

```bash
npm run lint
```

### Автоматичне виправлення помилок:

Якщо лінтер знаходить помилки, які можна виправити автоматично (наприклад, пропущені крапки з комою або неправильні відступи), виконайте:

```bash
npm run lint:fix
```

### Форматування коду (Prettier):

Щоб автоматично привести весь код до єдиного стилю (лапки, пробіли, довжина рядків):

```bash
npm run format
```

## 2. Конфігурації

- **.eslintrc.js**: Містить правила перевірки (використовується плагін для TypeScript та React Native).
- **.prettierrc**: Налаштування форматування (одинарні лапки, крапки з комою).

## 3. Поради

- Перед кожним комітом (commit) рекомендується запускати `npm run lint:fix` та `npm run format`.
- Використовуйте розширення ESLint та Prettier у вашому редакторі (наприклад, VS Code) для перевірки в реальному часі.
