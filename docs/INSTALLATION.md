# Інструкція з розгортання та запуску (Velder-soft)

Цей проект побудований на базі **Expo** (React Native) та **Firebase**. Він підтримує роботу на iOS, Android та у Веб-браузері.

## 1. Попередні вимоги

- Встановлений **Node.js** (LTS версія).
- Встановлений **npm** або **yarn**.
- Додаток **Expo Go** на вашому смартфоні (для тестування мобільної версії).

## 2. Налаштування Firebase

Проект використовує Firebase як Backend-as-a-Service.

1. Створіть проект у [Firebase Console](https://console.firebase.google.com/).
2. Увімкніть наступні сервіси:
   - **Authentication:** Email/Password.
   - **Firestore Database:** Створіть базу в тестовому режимі.
   - **Storage:** Створіть сховище для файлів.
3. Створіть **Web App** у налаштуваннях проекту Firebase та отримайте ключі конфігурації.

## 3. Налаштування оточення

1. Скопіюйте файл `.env.example` у новий файл `.env`:
   ```bash
   cp .env.example .env
   ```
2. Вставте ваші ключі Firebase у файл `.env`.

## 4. Встановлення залежностей

Виконайте команду в корені проекту:

```bash
npm install
```

## 5. Запуск додатка

### Для мобільної розробки (Android/iOS):

```bash
npx expo start
```

- Після запуску відскануйте QR-код через додаток **Expo Go** (Android) або камеру (iOS).

### Для веб-версії (ПК Директора):

```bash
npm run web
```

Додаток відкриється у вашому браузері за адресою `http://localhost:8081`.

## 6. Ролі користувачів

Додаток розділяє функціонал для **Директора** та **Працівника**.
Роль визначається у колекції `users` у Firestore.

- Створіть користувача через Auth.
- У Firestore створіть документ у колекції `users` з ID, що відповідає `uid` користувача.
- Додайте поле `role` зі значенням `DIRECTOR` або `EMPLOYEE`.
